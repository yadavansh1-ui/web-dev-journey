## 🧱 Semantic Tags (Page Structure)

Semantic tags define different parts of a webpage:

- `<header>` → Top section (title, intro)
- `<nav>` → Navigation links
- `<main>` → Main content
- `<section>` → Group of related content
- `<article>` → Independent content
- `<aside>` → Sidebar content
- `<footer>` → Bottom section

💡 These tags improve readability and make code look professional.

---

## 🧰 Utility Tags

### 🔹 Line Break
`<br>` → Moves content to next line  

Example:
```
Line 1<br>Line 2
```

---

### 🔹 Horizontal Line
`<hr>` → Creates a horizontal divider  

Example:
```
<hr>
```

---

## ✨ Text Formatting Tags

- `<strong>` → Important (bold)
- `<em>` → Emphasized text
- `<mark>` → Highlighted text
- `<small>` → Smaller text

💡 These tags add meaning, not just style.

---

## 🔬 Special Tags

- `<sub>` → Subscript (H₂O)
- `<sup>` → Superscript (10²)
- `<time>` → Represents date/time

Example:
```
H<sub>2</sub>O
10<sup>2</sup>
<time>2026</time>
```

---

## 🧠 Why Semantic HTML Matters

- Improves code readability  
- Helps SEO (search engines)  
- Improves accessibility  
- Used in real-world projects  

---

## 🚀 Extra Insight

Instead of writing:
```
<div><div><div>
```

Use:
```
<header> <main> <section>
```

👉 This is how professional developers write code.
